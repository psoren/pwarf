# Event System

> **Status:** Implemented
> **Last verified:** 2026-03-25

## Overview

Events are the narrative backbone of the game. Every notable thing that happens in the world is recorded as a `world_event` in the database and as a typed `SimEvent` flowing through the sim. Events feed the activity log, the legends screen, ruin histories, and dwarf memories.

## World Events (Database)

Stored in the `world_events` table:

```sql
world_events (
  id, world_id, year, category,
  civilization_id, ruin_id, dwarf_id, item_id, faction_id, monster_id,
  description,    -- human-readable text for the log
  event_data      -- JSONB with structured details
)
```

### Event Categories

14 categories stored as the `event_category` enum:

| Category           | Trigger                                         |
|-------------------|-------------------------------------------------|
| `battle`          | Combat between dwarves and monsters              |
| `death`           | A dwarf dies (any cause)                         |
| `birth`           | A dwarf is born in the fortress                  |
| `marriage`        | Two dwarves marry                                |
| `artifact_created`| A dwarf creates a masterwork/artifact            |
| `artifact_lost`   | An artifact is destroyed or stolen               |
| `fortress_founded`| Player embarks and founds a new civilization     |
| `fortress_fallen` | Civilization collapses                           |
| `migration`       | New dwarves arrive at the fortress               |
| `discovery`       | A new tile, cavern, or resource is discovered    |
| `myth`            | Legendary events (generated for flavor)          |
| `monster_sighting`| A new monster appears in the world               |
| `monster_slain`   | A monster is killed                              |
| `monster_siege`   | A monster begins attacking a fortress            |

### Indexed Queries

- `(world_id, year)` — timeline queries
- `category` — filter by type
- `monster_id` — monster history
- `description` GIN trigram index — full-text search across event descriptions

## Sim Events (Runtime)

Defined in `shared/src/events.ts` as a discriminated union (`SimEvent`). Each event type carries a `type` field plus event-specific data.

20 event types organized by domain:

### Tick
- `sim_tick` — heartbeat every step (for debugging/monitoring)

### Dwarf
- `dwarf_action` — a dwarf performs a job action (mine, build, craft, etc.)
- `dwarf_need_critical` — a need drops dangerously low
- `dwarf_tantrum` — a dwarf enters tantrum state
- `dwarf_mood` — strange mood, fey mood, possession, melancholy, berserk

### Combat
- `combat` — a single attack/defense exchange
- `combat_result` — overall encounter outcome

### Death
- `death` — a dwarf dies, with cause and optional killer

### Migration
- `migration` — new dwarves arrive

### Monster
- `monster_spawn` — a new monster appears
- `monster_move` — a monster changes tile
- `monster_siege` — a monster begins a siege
- `monster_slain` — a monster is killed

### Fortress
- `fortress_founded` — new civilization started
- `fortress_fallen` — civilization collapsed

### World
- `discovery` — new area/resource found
- `artifact_created` — masterwork item crafted
- `trade_caravan_arrival` — trading caravan arrives
- `disease_outbreak` — plague begins

### Annual
- `year_rollup` — annual summary (population, wealth, age)

## Event Flow

```
Phase produces event
       │
       ▼
Queued in SimContext.state.worldEvents
       │
       ▼
Phase 10 (event-firing) writes to Supabase world_events table
       │
       ▼
Supabase Realtime pushes to subscribed frontends
       │
       ▼
Frontend activity log / legends screen updates
```

## Activity Log Style

The log should feel like Dwarf Fortress's combat log: dense, specific, slightly literary.

Examples:
- "Urist McPickaxe strikes the Forgotten Beast in the left eye with a *steel battle axe*!"
- "Kadol Bronzehammer has been found dead. She was killed by Goblins."
- "Migrants have arrived! 4 dwarves have joined your fortress."
- "Doren Coalheart has created a masterwork *basalt throne*!"
- "The titan Storlut Redshadow has been sighted to the north."

Descriptions are stored in the `description` column of `world_events`. They should be generated by the sim with enough detail to be interesting when read in the legends screen years later.
